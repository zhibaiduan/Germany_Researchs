## Current State

The SAP Research site is now a **hash-based SPA** served from a single `index.html` shell. All routing is client-side via `assets/router.js`.

**What works:**
- `http://localhost:8080` (python3 http.server running on port 8080) loads the home page with topic grid
- Clicking a published topic card navigates to `#/topics/<slug>` — the article is fetched, parsed, and injected without a page reload
- Sidebar shows correct active state for current route (home vs. article)
- Sidebar is hidden on home page, visible on article pages (via `has-sidebar` CSS class toggle)
- TOC panel (collapsible dot-based, hover to expand) builds correctly after article load
- Sticky nav title appears when article header scrolls out of view
- Scroll dimming on TOC panel works
- Cleanup on route change: scroll listeners removed, `IntersectionObserver` disconnected, nav title removed, page-scoped styles purged
- Two published articles: `phase1-profit-model` and `company-overview`
- Fetch error state shows helpful message with `python3 -m http.server 8080` instructions (relevant because `fetch()` is blocked on `file://` protocol)

**Known constraint:** Must be served via HTTP — direct `file://` access breaks `fetch()`. Server must be running.

## Decisions Made

### Hash-based routing (not History API)
Chose `#/` over `/` + `pushState` because: no server-side routing config needed, works with `python3 -m http.server`, zero build tooling. Trade-off: ugly URLs. Acceptable for a personal research site.

### DOMParser extraction pattern for articles
Each article is a full standalone HTML file. The router fetches it, uses `DOMParser` to extract `.article-wrap` content (without executing scripts), injects page-specific `<style>` blocks into `<head>` with `data-page=slug` attribute for cleanup, then renders into `#view`. This keeps articles backward-compatible as standalone files while also working in SPA mode.

### Cleanup via passed-by-reference arrays
`buildFloatingTOC(scrollHandlers, onObserver)` and `initNavTitle(scrollHandlers)` push their listener function refs into the shared `_scrollHandlers` array. `cleanupView()` iterates and removes all of them. Same pattern for `IntersectionObserver` via callback. Clean, no globals leaked between routes.

### `renderHub()` returns HTML string (not DOM mutation)
Previously `renderHub()` directly mutated named DOM elements (`getElementById`). Refactored to return a full HTML string injected into `#view`. This is required for SPA — the hub elements don't exist until the router renders them.

### Sidebar hidden on home, visible on articles
User explicitly said "首页不需要那个目录结构". Router adds `has-sidebar` on article routes, removes it on home. CSS `has-sidebar` class controls flex layout and sidebar visibility.

## Open Questions

- **Deploy target?** If this goes beyond local, needs a static host (Netlify, GitHub Pages, Vercel). GitHub Pages works natively with hash routing. History API would require a redirect rule.
- **Should articles be their own standalone files or move content fully into `config.js`?** Current hybrid (standalone HTML files fetched by router) works well but means article content lives outside the config-driven system. Fine for long-form research articles; would be a pain if there were 50+ short entries.
- **Should the sidebar show on the home page?** Currently hidden per user request. If the topic list grows, having sidebar navigation on home could be useful. Easy to re-enable by adding `has-sidebar` in `showHome()`.
