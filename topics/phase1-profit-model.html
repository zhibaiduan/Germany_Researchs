专题 P1M1 · Profit & Strategy Transformation

盈利结构、客户迁移与 AI/Data 战略转型

SAP 正在如何从传统 ERP 公司转向云化和 AI/Data 时代的平台型公司

2026-05-14 18 分钟阅读

研究目的

这篇报告围绕一个核心问题展开：

从收入、成本、利润、地区分布和客户迁移来看，SAP 正在如何从传统 ERP 公司转向云化和 AI/Data 时代的平台型公司？

这个问题可以拆成四层，对应本文的四个主要章节：

01
事实层 — SAP 的钱从哪里来、花到哪里去、利润来自哪里，哪些地区贡献最大？
02
变化层 — 这些收入和利润结构为什么变化，是 SAP 主动战略调整，还是客户、市场和技术环境倒逼？
03
客户层 — 客户是否愿意按 SAP 的节奏从旧 ERP / support 模式迁到 cloud subscription？
04
判断层 — SAP 能否从 cloud ERP 公司进一步变成 enterprise AI + data platform 公司？

SAP 不只是单个软件公司，它是德国乃至欧洲 B2B 软件生态中的重要基础设施。SAP 的产品方向、迁云策略、AI 和数据平台布局、客户迁移节奏，都会影响咨询公司、系统集成商、垂直 SaaS、AI 初创、数据平台和企业 IT 预算流向。理解 SAP 的盈利结构和迁移机制，是判断德国 B2B 软件生态机会和风险的基础。

核心结论

以下五条结论贯穿全文，先提炼出来，方便读者在阅读细节时保持定位感：

C1
SAP 的收入中心已经从 license 转向 cloud。 2025 年，Cloud 已占总收入 57.1%，Software License 只剩 2.7%。主收入引擎已经从传统 license/support 转向 Cloud ERP Suite 驱动的订阅收入。
C2
传统 support 仍是现金牛，但正在被云转型逐步侵蚀。 Software Support 2025 年仍贡献 28.6% 收入，但已从 2023 年的 EUR 11.496B 降到 EUR 10.525B。SAP 在用 cloud 增长接住 support 和 license 的下滑。
C3
SAP 的目标不是简单"卖云"，而是控制企业流程、数据和 AI 执行层。 SAP 正在把 Cloud ERP Suite、Business Data Cloud、BTP、Joule / Business AI 组合起来，试图从 ERP 系统供应商变成企业流程 + 数据 + AI 执行层平台。
C4
客户迁移不是无阻力推进，而是持续谈判。 SAP 通过 RISE / GROW、旧系统维护期限、cloud-only innovation、AI/data 产品绑定推动迁移。但 DACH 大企业仍担心迁移成本、public cloud 标准化、长期合同成本和供应商锁定。
C5
第一阶段 cloud 转型基本成立，第二阶段 AI/Data 平台化仍待验证。 Cloud 收入、Cloud ERP Suite 占比和 Cloud 毛利率都支持转型叙事。但 AI/Data 产品的收入贡献尚未被单独量化，客户是否愿意把未来 AI 和数据入口交给 SAP，还没有完全证明。

一、事实层：SAP 的钱从哪里来、花到哪里去？

这一部分只回答事实问题：SAP 的收入、地区、成本和利润结构长什么样。所有数据来源：SAP 2025 Integrated Report / Form 20-F。

定义说明

Cloud ERP Suite 不只等于 S/4HANA Cloud。SAP 官方披露中，它包括 SAP S/4HANA Cloud、SAP Business Technology Platform（BTP），以及部分 HR/payroll、spend management、commerce、business process transformation、working capital management 等核心解决方案。更准确的理解是：SAP 的云增长高度集中在围绕 ERP 展开的核心业务套件，而不是外围 SaaS 产品的零散增长。

关键发现

EMEA 保留更多传统收入，核心原因不是"欧洲客户不懂云"，而是大型企业的 SAP 系统经过十几年定制，和工厂、仓库、银行、税务、供应商系统深度连接。直接迁到标准化 public cloud，可能意味着重写流程、接口、权限、报表和组织习惯。大企业更常选择 private cloud 或 hybrid 路线逐步迁移。

二、变化层：为什么收入和利润结构会变？

前面的事实说明了"发生了什么"。这一部分回答"为什么会这样"：哪些变化来自 SAP 主动转型，哪些来自客户和市场环境变化。

2.1 内因：SAP 主动把增长中心转向 Cloud ERP Suite

SAP 主动做了三件事：

第一，把增长中心从 License 转到 Cloud ERP Suite。 License 收入从 EUR 1.764B 降到 EUR 0.990B，而 Cloud ERP Suite 从 EUR 10.626B 增到 EUR 18.119B。这不是自然变化，而是 SAP 主动把销售、产品和客户迁移都导向 cloud-first。

第二，用 RISE / GROW / Business Suite 设计迁移路径。

路径 目标客群 部署偏好 定位
RISE with SAP 已有 SAP on-prem 大型老客户 Private cloud / 复杂转型 迁移路径
GROW with SAP 新客户、成长型企业、流程标准化 Public cloud 迁移路径
SAP Business Suite 所有客户 Cloud 目标产品世界

RISE 和 GROW 是迁移/销售路径，Business Suite 是 SAP 希望客户最终进入的云产品版图。

第三，围绕 AI 和 Data 重组产品版图。 SAP 推出 Business Data Cloud，并与 Databricks 合作；2026 年完成 Reltio 收购，宣布收购 Dremio 和 Prior Labs。产品版图已经重组为四层：

应用层
业务应用系统
Cloud ERP、SuccessFactors、Ariba、Concur、Commerce、Supply Chain 等核心业务系统
数据层

Reltio、Dremio、Databricks 合作等，打通 SAP 和非 SAP 数据，形成可治理业务数据层
平台层

AI agents，把数据和流程转化为建议、自动化动作和业务执行

2.2 外因：客户、AI 和资本市场也在逼 SAP 改变

SAP 的外部压力不是"马上被替代"的生存危机。它的 ERP 深度嵌入大型企业核心流程，客户切换成本极高。真正的压力是：如果 SAP 不把客户带到 cloud、data 和 AI 平台上，它会逐渐失去未来企业软件的控制点。

外因 影响程度 对 SAP 的具体影响
旧 ERP 客户进入换代周期 高 直接推动客户从旧 SAP Business Suite / ECC 迁向 S/4HANA 和 cloud；Business Suite 7 主流维护到 2027 年底，extended maintenance 到 2030 年底
AI 需要云和数据底座 高 决定 SAP 能否在 AI 时代继续掌握企业流程和数据入口
竞争者蚕食外围模块 中高 Workday、Salesforce、ServiceNow、Microsoft、Databricks、Snowflake 等削弱 SAP 在 HR、CRM、workflow、data platform、AI layer 的控制点
资本市场偏好可预测收入 中 强化 SAP 转向 cloud subscription 的动力；2025 年"more predictable revenue"占比达 86%，高于 2021 年的 75%
综合判断

SAP 的转型既是主动寻找新增长，也是防守核心控制点。短中期最直接的压力来自旧 ERP 换代周期，长期最重要的压力来自 AI 时代对数据和流程控制权的重新争夺。

三、客户层：迁移是否顺利？

SAP 的云化不是单向推进，而是一场和客户之间的持续谈判。客户仍然承认 SAP 的重要性，但不愿无条件跟随 SAP 的节奏迁移。

关键是最后两个数字：62% 的客户对 SAP 的新 Business Suite 愿景没有积极跟进。在 S/4HANA 投资偏好中，on-prem 仍远高于 public cloud（高/中等投资计划：on-prem 42%、private cloud / RISE 22%、public cloud / GROW 仅 6%）。

客户仍然承认 SAP 是核心基础设施，但持续追问四个现实问题：

Q1
迁移到 cloud 后，原有定制、接口和行业流程能否保留？
Q2
新的 AI/data 产品能否带来明确 ROI？
Q3
订阅合同的长期成本是否可控？
Q4
进入 SAP cloud 后，客户议价权和退出选择是否会变弱？

3.2 SAP 的迁移策略：激励 + 压力

SAP 的迁移策略可以概括为 "激励 + 压力"：一方面让 cloud 看起来更有价值，另一方面让留在旧模式里的成本和风险逐步上升。

激励 — 云作为未来创新入口
  • RISE with SAP 降低大型老客户从 on-prem 到 cloud 的迁移阻力
  • GROW with SAP 给新客户和标准化客户更快上线的 public cloud ERP 路径
  • Business Suite 把 Cloud ERP、Business Data Cloud、BTP、Business AI 组合成统一产品叙事
  • Joule、Business AI、Business Data Cloud 等新能力更多放在 cloud 体系里
  • LeanIX、WalkMe、Reltio、Dremio、Prior Labs 等收购，分别补迁移理解、用户采用、主数据、数据湖仓和结构化 AI 能力
压力 — 旧模式越来越不舒服
  • SAP Business Suite 7 主流维护到 2027 年底，extended maintenance 到 2030 年底
  • 产品创新重点越来越集中到 cloud、Business Suite、Business AI 和 Business Data Cloud
  • SAP 的销售、合同和投资者叙事都在强化 cloud subscription 和可预测收入
  • 咨询公司、系统集成商、实施伙伴也逐渐围绕 RISE / GROW / Business Suite 重组

SAP 不是只靠说服客户"喜欢云"，而是在重塑客户的选择空间：留在旧系统越来越难获得创新，迁移到 SAP cloud 才能进入新的产品、数据和 AI 体系。

3.3 定价模式：从 license/support 到 cloud subscription

SAP 的商业模式变化，本质是从 一次性 license + 年度 support 转向 长期 cloud subscription + 平台和 AI/data 附加价值。

维度 传统模式 云模式
核心收费 一次性 software license 按订阅周期持续付费（FUE 口径）
附加收费 年度 support / maintenance BTP、Business Data Cloud、AI agents 等附加能力
对 SAP license 收入波动大；support 稳定高毛利 收入更可预测，客户绑定更深，持续扩展空间
对客户 一次性大额支出，后续 support 成本清晰 长期总成本不确定，议价权可能下降，退出成本更高

3.4 阶段判断：财务转型顺利，客户迁移不完全顺利

财务层面 · 较顺利

  • Software license 下滑被 Cloud 增长覆盖

  • AI/Data 产品真实收入贡献尚未单独量化披露

SAP 的财务转型顺利，客户迁移体验不完全顺利；收入结构已经变了，但客户心智和系统架构还在过渡期。

四、判断层：SAP 在 AI/Data 时代的位置

判断 SAP 这轮转型，不能只问"cloud 收入有没有增长"。更重要的是看：SAP 能否从 cloud ERP 公司进一步变成 enterprise AI + data platform 公司。

4.1 SAP 想控制什么？

SAP 的目标，是从 ERP 软件供应商 变成 企业流程 + 数据 + AI 执行层平台，这套目标可以拆成四个控制点：

控制点 01
流程控制点
SAP 已经深度嵌入企业财务、采购、供应链、生产、库存、人力等核心流程。
控制点 02
数据控制点
SAP 想通过 Business Data Cloud，把 SAP 和非 SAP 数据整理成可治理、可理解、可调用的业务数据层。
控制点 03
平台控制点
SAP 用 BTP 承担集成、扩展、自动化和 AI 接入，是生态系统的技术枢纽。
控制点 04
AI 执行控制点
SAP 用 Joule、Business AI、AI agents，把 AI 从"回答问题"推进到"理解业务流程并执行动作"。
战略定位

SAP 的战略不是和 OpenAI、Anthropic、Google 正面竞争通用大模型，而是试图成为企业 AI 落地时连接流程、数据、权限和执行的业务层。企业 AI 最缺的不是模型本身，而是可靠的业务上下文、干净的数据、权限体系、流程约束和可执行入口。SAP 恰好掌握了很多大型企业最核心的业务流程和历史数据。

4.2 好信号与坏信号

好的信号
  • Cloud 转型在财务上已经成立，Cloud 已经成为最大收入来源
  • Cloud ERP Suite 占 Cloud 收入 86.2%，云增长围绕核心业务套件推进
  • Software license 下滑没有拖垮总收入，cloud 增长能够覆盖旧模式收缩
  • Cloud 毛利率约 73.9%，已进入规模化盈利阶段
  • LeanIX、WalkMe、Reltio、Dremio、Prior Labs 等动作，正在补完关键能力拼图
  • 大型企业客户基础仍是护城河，核心 ERP 不容易被快速替代
坏的信号
  • 客户迁移体验仍然有阻力，尤其 DACH 大企业对 public cloud 并不完全接受
  • Public cloud 标准化与大企业复杂定制需求存在张力
  • AI/Data 的收入贡献尚未清晰披露，仍更多是下一阶段增长假设
  • Microsoft、Databricks、Snowflake、ServiceNow、Salesforce、Workday、Oracle 都在争夺企业数据、workflow 和 AI agent 入口
  • 产品版图很大，把 Cloud ERP、BTP、Business Data Cloud、Joule 等真正整合成低摩擦客户体验，难度很高

4.3 总体判断

第一阶段 · 大概率成功
从 license/support 向 cloud subscription 的转型
  • 财务数据支撑：Cloud 占比 57.1%
  • Cloud ERP Suite 规模验证（86.2% 集中度）
  • Cloud 毛利率 73.9%，达到成熟水平
  • 总收入稳健增长（2023–2025 +17.9%）
第二阶段 · 方向对，待验证
从 cloud ERP 变成 enterprise AI + data platform
  • AI/Data 收入贡献尚未单独量化
  • 客户是否愿意把 AI 和数据入口交给 SAP
  • 产品整合的低摩擦体验仍需证明
  • 竞争对手都在争夺同一个入口

SAP 在 AI 时代最大的竞争力来自它对企业核心流程和业务数据的控制；最大的风险也来自同一个地方：这些流程和系统太复杂，导致客户迁移慢、标准化难、AI/data 产品落地周期长。

SAP 未来竞争力的关键，不是它能不能发布更多 AI 功能，而是它能否把 Cloud ERP Suite、Business Data Cloud、BTP 和 Joule 真正组合成一个客户愿意迁移、愿意付费、能够产生业务结果的统一平台。
